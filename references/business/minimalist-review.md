## 极简创业者审查

**核心原则：** 用极简创业者的思维框架检验任何商业决策。

**宣布：** "我正在使用 minimalist-review 技能来审查这个决策。"

## 极简审查清单

对任何商业决策，问以下问题：

### 1. 社区导向
- [ ] 这服务于我的社区吗？
- [ ] 这是社区要求的还是我觉得他们需要的？
- [ ] 我有来自社区的真实反馈支持这个决策吗？

### 2. 盈利优先
- [ ] 这从第一天就能盈利吗？
- [ ] 如果不能立即盈利，多久能回本？
- [ ] 最坏的情况下我能承受多少损失？

### 3. 极简原则
- [ ] 这是最简单的方案吗？
- [ ] 有没有更简单的替代方案？
- [ ] 我是在解决真实问题还是在过度工程化？

### 4. 可持续性
- [ ] 我能独自维护这个吗？
- [ ] 这增加还是减少了我的工作量？
- [ ] 6 个月后我还愿意做这件事吗？

### 5. 可逆性
- [ ] 这个决策是可逆的吗？
- [ ] 如果失败了，恢复成本是多少？
- [ ] 我能用低成本测试这个决策吗？

## 判决模板

```markdown
## 极简审查：[决策描述]

**社区导向:** ✅/❌ — [说明]
**盈利优先:** ✅/❌ — [说明]
**极简原则:** ✅/❌ — [说明]
**可持续性:** ✅/❌ — [说明]
**可逆性:** ✅/❌ — [说明]

**判决:** GO / NO-GO / 需要更多验证
**理由:** [一段话总结]
**如果 GO，下一步:** [具体行动]
```

## 压力测试

### 高压场景
- 每周都在加功能，但看不清哪些值得保留。

### 常见偏差
- 把“忙”误认为“有效进展”。

### 使用技能后的纠正
- 按 ROI 和复杂度复盘，删掉不能带来复利的项目。
