# Context Budget Rules — 上下文预算规则

保持编排器上下文精简的标准规则。

---

## 通用规则

1. **永不**读取代理定义文件 — subagent 自动加载
2. **永不**将大文件内联到代理提示 — 让代理从磁盘读
3. **读取深度随上下文窗口缩放**
4. **委托**重活给子代理
5. **主动警告**：消耗大量上下文时提醒用户

## 上下文退化层级

| 层级 | 使用率 | 行为 |
|------|--------|------|
| **PEAK** | 0-30% | 全功能。读全文、生成多代理、内联结果 |
| **GOOD** | 30-50% | 正常操作。偏好读前言，积极委托 |
| **DEGRADING** | 50-70% | 节约模式。仅读前言，最小内联，警告用户 |
| **POOR** | 70%+ | 紧急模式。立即检查点。仅关键读取 |

## 退化警告信号

- **静默部分完成** — 代理声称完成但实现不完整
- **越来越模糊** — 代理开始用"适当处理"代替具体代码
- **跳过步骤** — 代理省略正常应遵循的协议步骤

## 一人公司上下文策略

一人公司创始人频繁切换上下文（编码→营销→客服→编码）。SuperOPC 通过以下方式管理：

1. **会话摘要** — 每次会话结束时持久化状态
2. **技能缓存** — 频繁使用的技能保持在上下文中
3. **代理隔离** — 每个代理有独立上下文窗口，防止上下文腐蚀
4. **检查点机制** — 长任务定期保存进度，可从中断处恢复
