# 路线图：[项目名称] — API Service

## 概览

从项目骨架到生产就绪的 API 服务：基础设施 → 认证 → 业务 API → 部署。

## 阶段

- [ ] **阶段 1：基础设施** - 项目骨架 + 数据库 + 中间件
- [ ] **阶段 2：认证授权** - JWT + 用户管理 + 角色
- [ ] **阶段 3：业务 API** - CRUD + 验证 + 文档
- [ ] **阶段 4：部署上线** - Docker + CI/CD + 监控

## 阶段详情

### 阶段 1：基础设施
**目标**：可运行的项目骨架，数据库已连接
**依赖**：无
**需求**：INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05, DB-01, DB-02
**成功标准**：
  1. `dotnet run` 可以启动，Swagger UI 可访问
  2. 数据库迁移可以成功执行
  3. Health Check 端点返回 200
  4. 错误请求返回 RFC 7807 格式

### 阶段 2：认证授权
**目标**：用户可以注册、登录、Token 刷新
**依赖**：阶段 1
**需求**：AUTH-01, AUTH-02, AUTH-03, AUTH-04, AUTH-05
**成功标准**：
  1. 注册端点可以创建用户
  2. 登录返回有效 JWT
  3. 受保护端点拒绝无 Token 请求
  4. Refresh Token 可以换取新 Access Token

### 阶段 3：业务 API
**目标**：核心业务端点可用，文档完善
**依赖**：阶段 2
**需求**：API-01, API-02, API-03, API-04, API-05, DOC-01, DOC-02, DOC-03, DB-03
**成功标准**：
  1. 所有 CRUD 端点通过集成测试
  2. 非法请求返回具体的验证错误
  3. 分页查询返回正确的总数和数据
  4. Swagger 文档包含所有端点的示例

### 阶段 4：部署上线
**目标**：API 服务可以安全地对外提供
**依赖**：阶段 3
**成功标准**：
  1. Docker 镜像可以构建并运行
  2. CI/CD 流水线：测试 → 构建 → 部署
  3. 生产环境 Health Check 正常
  4. 安全审计通过（无硬编码密钥）
