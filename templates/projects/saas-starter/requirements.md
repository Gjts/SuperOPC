## v1 需求

### 认证 (AUTH)

- [ ] **AUTH-01**: 用户可以用邮箱和密码注册
- [ ] **AUTH-02**: 用户注册后收到邮箱验证
- [ ] **AUTH-03**: 用户可以通过 Google/GitHub OAuth 登录
- [ ] **AUTH-04**: 用户可以通过邮件链接重置密码
- [ ] **AUTH-05**: 登录状态通过 HttpOnly Cookie 保持

### 仪表盘 (DASH)

- [ ] **DASH-01**: 登录后看到核心指标概览
- [ ] **DASH-02**: 可以创建/编辑/删除核心资源
- [ ] **DASH-03**: 数据列表支持分页和搜索
- [ ] **DASH-04**: 响应式布局适配移动端

### 支付 (PAY)

- [ ] **PAY-01**: 展示定价页面（Free / Pro / Enterprise）
- [ ] **PAY-02**: 用户可以通过 Stripe Checkout 订阅 Pro 方案
- [ ] **PAY-03**: 用户可以通过 Customer Portal 管理订阅
- [ ] **PAY-04**: Webhook 同步订阅状态到数据库
- [ ] **PAY-05**: 根据订阅级别限制功能访问

### 设置 (SET)

- [ ] **SET-01**: 用户可以更新个人信息（名称/头像）
- [ ] **SET-02**: 用户可以更改密码
- [ ] **SET-03**: 用户可以删除账户（GDPR 合规）

### 营销页 (MKT)

- [ ] **MKT-01**: Landing page 展示价值主张和 CTA
- [ ] **MKT-02**: SEO meta tags 和 Open Graph 标签
- [ ] **MKT-03**: 页脚包含隐私政策和服务条款链接

## v2 需求

### 团队

- **TEAM-01**: 邀请团队成员
- **TEAM-02**: 角色权限（管理员/成员/只读）

### 集成

- **INT-01**: API 密钥管理
- **INT-02**: Webhook 出站通知

### 分析

- **ANL-01**: 使用量仪表盘
- **ANL-02**: 导出数据为 CSV
