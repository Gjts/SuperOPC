# 路线图：[项目名称] — SaaS

## 概览

从零到可收费的 SaaS MVP：认证 → 核心功能 → 支付 → 营销页 → 上线。

## 阶段

- [ ] **阶段 1：认证系统** - Supabase Auth + 用户管理
- [ ] **阶段 2：核心功能** - 仪表盘 + CRUD + 数据模型
- [ ] **阶段 3：支付集成** - Stripe Checkout + 订阅管理
- [ ] **阶段 4：营销与 SEO** - Landing page + 定价页 + SEO
- [ ] **阶段 5：上线准备** - 部署 + 监控 + 最终验证

## 阶段详情

### 阶段 1：认证系统
**目标**：用户可以注册、登录、重置密码
**依赖**：无
**需求**：AUTH-01, AUTH-02, AUTH-03, AUTH-04, AUTH-05
**成功标准**：
  1. 新用户可以完成注册并收到验证邮件
  2. 用户可以通过 Google/GitHub 一键登录
  3. 忘记密码流程可以正常重置
  4. 未登录用户访问受保护页面会被重定向

### 阶段 2：核心功能
**目标**：用户可以使用产品核心功能
**依赖**：阶段 1
**需求**：DASH-01, DASH-02, DASH-03, DASH-04, SET-01, SET-02, SET-03
**成功标准**：
  1. 登录用户可以看到个人仪表盘
  2. 可以创建、编辑、删除核心资源
  3. 数据列表支持分页和搜索
  4. 在手机浏览器上布局不崩溃

### 阶段 3：支付集成
**目标**：用户可以订阅付费方案
**依赖**：阶段 2
**需求**：PAY-01, PAY-02, PAY-03, PAY-04, PAY-05
**成功标准**：
  1. 定价页清晰展示方案对比
  2. 点击升级可以跳转 Stripe Checkout 完成支付
  3. 支付成功后用户权限立即升级
  4. 用户可以通过 Customer Portal 取消或变更订阅

### 阶段 4：营销与 SEO
**目标**：Landing page 可以转化访客为注册用户
**依赖**：阶段 3
**需求**：MKT-01, MKT-02, MKT-03
**成功标准**：
  1. Landing page Lighthouse Performance > 90
  2. 所有页面有正确的 meta tags
  3. 隐私政策和服务条款页面存在

### 阶段 5：上线准备
**目标**：产品可以安全地对外发布
**依赖**：阶段 4
**成功标准**：
  1. 生产环境部署到 Vercel + 自定义域名
  2. 错误监控（Sentry）已配置
  3. 数据库备份策略已确认
  4. 安全审计通过（无硬编码密钥、RLS 已启用）
