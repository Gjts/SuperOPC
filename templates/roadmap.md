# ROADMAP.md 模板

> 用于 `.opc/ROADMAP.md` — 项目阶段路线图和状态追踪。

---

## 文件模板

```markdown
# 路线图：[项目名称]

## 概览

[一段话描述从开始到完成的旅程]

## 阶段

**阶段编号：**
- 整数阶段（1, 2, 3）：已规划的里程碑工作
- 小数阶段（2.1, 2.2）：紧急插入（标记 INSERTED）

- [ ] **阶段 1：[名称]** - [一句话描述]
- [ ] **阶段 2：[名称]** - [一句话描述]
- [ ] **阶段 3：[名称]** - [一句话描述]

## 阶段详情

### 阶段 1：[名称]
**目标**：[此阶段交付什么]
**依赖**：无（首个阶段）
**需求**：[REQ-01, REQ-02, REQ-03]
**成功标准**（必须为真）：
  1. [从用户角度的可观察行为]
  2. [从用户角度的可观察行为]
**计划数**：[数量，如 "3 个计划" 或 "待定"]

计划：
- [ ] 01-01：[第一个计划的简要描述]
- [ ] 01-02：[第二个计划的简要描述]

### 阶段 2：[名称]
**目标**：[此阶段交付什么]
**依赖**：阶段 1
**需求**：[REQ-04, REQ-05]
**成功标准**（必须为真）：
  1. [从用户角度的可观察行为]
**计划数**：[数量]

计划：
- [ ] 02-01：[简要描述]

## 进度

| 阶段 | 已完成计划 | 状态 | 完成日期 |
|------|-----------|------|---------|
| 1. [名称] | 0/3 | 未开始 | - |
| 2. [名称] | 0/2 | 未开始 | - |
| 3. [名称] | 0/2 | 未开始 | - |
```

---

## 使用指南

### 粒度配置
阶段数量取决于 config.json 中的粒度设置：
- **coarse（粗）**：3-5 个阶段，每个 1-3 个计划
- **standard（标准）**：5-8 个阶段，每个 3-5 个计划
- **fine（细）**：8-12 个阶段，每个 5-10 个计划

### 成功标准
- 每阶段 2-5 个可观察行为
- 从用户视角描述
- 格式："用户可以 [操作]" 或 "[事物] 存在/工作正常"
- 在路线图创建时与需求交叉检查

### 状态值
- `未开始` — 尚未开始
- `进行中` — 当前正在工作
- `完成` — 已完成（添加日期）
- `延期` — 推迟到以后（附原因）

### 里程碑后重组
完成首个里程碑后，用 `<details>` 标签折叠已完成阶段：

```markdown
<details>
<summary>✅ v1.0 MVP（阶段 1-4）- 已发布 YYYY-MM-DD</summary>
[已完成的阶段详情...]
</details>
```
