# Community Guide | 社区指南

> SuperOPC 社区的参与指南和最佳实践

## 社区渠道

### GitHub Discussions

GitHub Discussions 是我们的主要社区平台，用于：

| 类别 | 用途 |
|------|------|
| **Announcements** | 版本发布、重要更新 |
| **Ideas** | 新技能/代理/命令的提议 |
| **Q&A** | 使用问题和技术支持 |
| **Show and Tell** | 展示你用 SuperOPC 构建的项目 |
| **General** | 自由讨论 |

**如何参与：**
1. 访问 [SuperOPC Discussions](https://github.com/gjts/superopc/discussions)
2. 浏览现有讨论或创建新话题
3. 为你感兴趣的想法投票

### Discord

实时交流平台（即将开放）：

| 频道 | 用途 |
|------|------|
| `#general` | 日常交流 |
| `#help` | 技术支持 |

### 微信群

中文社区交流（即将开放）：
- 扫码加入 SuperOPC 中文社区
- 每周五晚上 8 点在线交流
- 不定期举办 AMA（Ask Me Anything）

## 社区活动

### 月度技能挑战

每月一个主题，社区成员共同创建和改进技能：
- 第 1 周：主题公布 + 讨论
- 第 2-3 周：创建和提交
- 第 4 周：评选和合并

### 季度复盘

每个季度回顾 SuperOPC 的发展：
- 版本进展回顾
- 社区贡献统计
- 下个季度路线图投票
- 贡献者表彰

### Show and Tell

鼓励社区成员分享：
- 用 SuperOPC 构建的产品
- 自定义的技能和工作流
- Building in Public 的经验
- 一人公司运营心得

## 获取帮助

### 提问最佳实践

1. **先搜索** — 在 Discussions 和 Issues 中搜索是否已有答案
2. **提供上下文** — 使用的 AI 工具、SuperOPC 版本、OS
3. **最小复现** — 提供复现问题的最小步骤
4. **附上输出** — 错误信息、日志或截图

### 响应时间预期

| 渠道 | 预期响应时间 |
|------|-------------|
| GitHub Issues (Bug) | 48 小时内 |
| GitHub Discussions | 72 小时内 |
| Discord | 实时（社区互助） |
| PR Review | 1 周内 |

## 行为规范

所有社区成员需遵守 [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md)。

核心原则：
- **尊重** — 每个人都是同行者
- **包容** — 欢迎所有背景的参与者
- **务实** — 聚焦实际价值
- **建设性** — 提出问题时也提供建议
