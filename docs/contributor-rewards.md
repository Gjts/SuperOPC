# Contributor Rewards | 贡献者奖励机制

> SuperOPC 感谢每一位贡献者——你的贡献让更多一人公司创始人受益

## 贡献者等级

| 等级 | 要求 | 认可方式 |
|------|------|---------|
| **Contributor** | 1+ 个被合并的 PR | README 署名 + Contributors 列表 |
| **Regular** | 3+ 个被合并的 PR | 以上 + Discord/微信核心贡献者角色 |
| **Core** | 10+ 个被合并的 PR 或维护一个子系统 | 以上 + 仓库 Write 权限 + 决策投票权 |
| **Maintainer** | 持续维护 + 社区领导 | 以上 + 仓库 Admin 权限 |

## 贡献类型与积分

| 贡献类型 | 等价 PR 数 | 说明 |
|---------|-----------|------|
| 新技能 | 1 | 通过质量审查的 SKILL.md |
| 新代理 | 1 | 通过质量审查的代理定义 |
| 新命令 | 0.5 | 通过质量审查的命令定义 |
| 新项目模板 | 2 | 完整的项目模板（README + .opc 配置） |
| Bug 修复 | 0.5-1 | 视复杂度 |
| 文档改进 | 0.5 | 翻译、修正、补充 |
| 新运行时支持 | 2 | convert.py 中添加新运行时 |
| 压力测试场景 | 0.5 | 为现有技能补充压力测试 |
| 社区帮助 | 0.5 | 在 Discussions 中帮助其他用户 |

## 特殊认可

### Hall of Fame / 荣誉殿堂

持续贡献并对项目产生显著影响的贡献者将被收入荣誉殿堂，包括：
- 在 README.md 和网站中的永久署名
- 社交媒体上的公开感谢
- 参与项目路线图决策

### First-Time Contributor | 首次贡献者

我们特别欢迎首次贡献者！标记为 `good first issue` 的 Issue 是最佳入门点：
- 修复文档中的错别字
- 为现有技能补充压力测试
- 翻译文档为其他语言
- 改进现有技能的触发条件描述

## 社区渠道

| 渠道 | 用途 | 链接 |
|------|------|------|
| **GitHub Discussions** | 功能讨论、问答、想法分享 | [Discussions](https://github.com/gjts/superopc/discussions) |
| **GitHub Issues** | Bug 报告、功能请求 | [Issues](https://github.com/gjts/superopc/issues) |
| **Discord** | 实时讨论、社区交流 | 即将开放 |
| **微信群** | 中文社区交流 | 即将开放 |

## 如何开始贡献

1. **阅读 CONTRIBUTING.md** — 了解贡献规范
2. **浏览 `good first issue`** — 找到适合入门的任务
3. **加入 Discussions** — 分享你的想法
4. **Fork & PR** — 提交你的贡献
5. **获得认可** — 被合并后自动获得贡献者身份

## 贡献者公约

所有贡献者需遵守 [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md)。

## 愿景

SuperOPC 是为一人公司创始人打造的工具。每一个贡献——无论是一个错别字修复还是一个全新的技能——都在帮助世界上某个正在独自奋斗的创始人。

**你的贡献，让一人公司不再孤单。**
