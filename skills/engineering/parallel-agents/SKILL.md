# 并行代理 — 波次执行引擎

将独立任务分组为波次，同一波次内并行派发代理，波次间严格依赖验证。

## 刚性规则

1. **波次内并行，波次间串行** — 同一波次的任务无依赖，可并行；下一波次依赖上一波次的产物
2. **波前验证** — 波次 N+1 启动前，必须验证波次 N 的所有产物存在且通过质量检查
3. **失败不传播** — 单个任务失败不影响同波次其他任务，但阻止依赖它的下游任务
4. **编排器不执行** — 编排器只做依赖分析和派发，实际工作由子代理完成

## 波次分析流程

### Step 1: 依赖图构建

从任务列表构建有向无环图（DAG）：

```
输入：任务列表 + 每个任务的依赖
输出：拓扑排序的波次分组

示例：
  A (无依赖)
  B (无依赖)
  C (依赖 A)
  D (依赖 A, B)
  E (依赖 C)

波次分组：
  Wave 1: [A, B]      — 无依赖，并行
  Wave 2: [C, D]      — 依赖 Wave 1，并行
  Wave 3: [E]         — 依赖 Wave 2
```

### Step 2: 波次执行

```
for each wave in waves:
  1. 验证前置波次产物（Pre-flight Gate）
  2. 为每个任务创建独立子代理上下文
  3. 并行派发所有子代理
  4. 等待所有子代理完成
  5. 收集结果，更新状态
  6. 质量检查（Revision Gate）
  7. 失败任务 → 重试（最多 2 次）或标记失败
```

### Step 3: 波前验证

波次 N+1 启动前检查：
- [ ] 波次 N 所有任务状态：completed 或 failed
- [ ] 依赖任务的产物文件存在
- [ ] 产物通过基本验证（非空、语法正确）
- [ ] 失败任务没有阻塞当前波次的任务

## 子代理上下文管理

每个子代理获得：
- **新鲜上下文** — 不继承编排器的上下文膨胀
- **精确任务描述** — 只给需要做的事
- **输入文件列表** — 告诉代理读哪些文件（不要内联）
- **输出规格** — 明确产物格式和位置

```
子代理上下文模板：
---
任务：[具体任务描述]
输入：[需要读取的文件列表]
输出：[预期产物的路径和格式]
约束：[时间/质量/安全要求]
---
```

## 失败处理

| 情况 | 行动 |
|------|------|
| 单任务失败（非阻塞） | 标记失败，继续其他任务 |
| 单任务失败（阻塞下游） | 重试 2 次，仍失败则暂停依赖链 |
| 波次 > 50% 任务失败 | 暂停整个执行，报告给用户 |
| 全部成功 | 推进到下一波次 |

## 一人公司适配

- **小波次** — 一人公司通常 2-4 个并行任务，不需要大规模并行
- **上下文预算** — 每个子代理独立上下文，避免编排器上下文爆炸
- **快速失败** — 早发现问题早停止，不浪费时间
- **可恢复** — 从失败的波次恢复，不需要从头开始
