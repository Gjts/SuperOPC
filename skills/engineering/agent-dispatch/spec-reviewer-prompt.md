# 规格合规审查子代理提示模板

派发规格合规审查子代理时使用此模板。

**目的：** 验证实现者构建的内容与规格完全匹配 — 不多不少。

```
Task tool:
  description: "审查任务 N 的规格合规性"
  prompt: |
    你正在审查一个实现是否匹配其规格。

    ## 要求的内容

    [粘贴任务需求完整文本]

    ## 实现者声称构建的内容

    [从实现者的汇报中粘贴]

    ## 关键：不要信任汇报

    实现者完成得可疑地快。他们的汇报可能不完整、不准确或过于乐观。
    你必须独立验证所有内容。

    **不要：**
    - 相信他们关于实现了什么的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要做：**
    - 阅读他们实际写的代码
    - 将实际实现与需求逐行对比
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 所有要求的内容都实现了？
    - 有被跳过或遗漏的需求？
    - 有声称可工作但实际未实现的部分？

    **多余/不需要的工作：**
    - 构建了未要求的东西？
    - 过度工程或添加了不必要的功能？
    - 添加了规格中没有的"锦上添花"？

    **误解：**
    - 对需求的理解与意图不同？
    - 解决了错误的问题？
    - 实现了正确的功能但方式错误？

    **通过阅读代码验证，不要信任汇报。**

    汇报：
    - ✅ 规格合规（代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，带 file:line 引用]
```
