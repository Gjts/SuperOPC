# 代码质量审查子代理提示模板

派发代码质量审查子代理时使用此模板。

**目的：** 验证实现质量 — 干净、经过测试、可维护。

**前置条件：** 仅在规格合规审查通过后才派发此审查。

```
Task tool:
  description: "审查任务 N 的代码质量"
  prompt: |
    你正在对一个已通过规格合规审查的实现进行代码质量审查。

    ## 实现内容

    [从实现者汇报中粘贴实现摘要]

    ## 计划/需求

    任务 N 来自 [计划文件]

    ## 变更范围

    BASE_SHA: [任务前的提交]
    HEAD_SHA: [当前提交]

    ## 审查重点

    除标准代码质量问题外，特别检查：

    **职责分离：**
    - 每个文件是否有一个清晰职责和明确接口？
    - 单元是否分解为可独立理解和测试的粒度？

    **计划一致性：**
    - 实现是否遵循计划中的文件结构？
    - 是否创建了已经很大的新文件？
    - 是否显著增长了现有文件？（不要标记已存在的文件大小问题）

    **测试质量：**
    - 测试验证行为而非实现细节？
    - 测试覆盖正常路径和边界情况？
    - 测试名称描述性强？

    **代码整洁：**
    - 命名准确反映意图？
    - 无魔法数字/字符串？
    - 无重复代码？
    - 错误处理完善？

    ## 汇报格式

    **优点：** [做得好的地方]

    **问题：**
    - 🔴 关键（必须修复）：[阻塞性问题]
    - 🟡 重要（应该修复）：[质量问题]
    - 🟢 轻微（可以修复）：[建议改进]

    **评估：** ✅ 通过 | ❌ 需要修复
```
