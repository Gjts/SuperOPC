# Security Guidelines — SuperOPC 通用安全规范

## 提交前强制检查

- [ ] 无硬编码密钥（API keys, passwords, tokens）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（HTML 净化）
- [ ] CSRF 防护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点启用速率限制
- [ ] 错误消息不泄露敏感数据

## 密钥管理

- **永不**在源码中硬编码密钥
- **始终**使用环境变量或密钥管理器
- 启动时验证所需密钥是否存在
- 轮换任何可能已暴露的密钥

## 安全响应协议

发现安全问题时：
1. **立即停止**
2. 使用 **opc-reviewer** 代理进行安全审查
3. 先修复**关键**问题，再继续
4. 轮换任何暴露的密钥
5. 审查整个代码库是否有类似问题

## OWASP Top 10 速查

| # | 风险 | 一人公司重点 |
|---|------|-------------|
| A01 | 访问控制失效 | 每个端点检查权限 |
| A02 | 加密失败 | 密码用 bcrypt/argon2，TLS 全程 |
| A03 | 注入 | 参数化查询，ORM 优先 |
| A05 | 安全配置错误 | 生产环境关闭调试模式 |
| A07 | 认证失败 | JWT + 刷新令牌，MFA |
| A09 | 日志监控不足 | 结构化日志 + 告警 |

## 一人公司安全优先级

时间有限时，按此顺序投入：
1. **认证和授权** — 不安全的 API 是最大风险
2. **输入验证** — 拦截注入和 XSS
3. **密钥管理** — 不要把密钥提交到 Git
4. **依赖更新** — `npm audit` / `dotnet list package --vulnerable`
5. **日志** — 出问题时你需要知道发生了什么
